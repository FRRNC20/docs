<Update label="July 13, 2026">

## ✨ Updates

**Mobile staff console — iOS-native polish**

The mobile staff console now feels like a first-class iOS app. This update layers native chrome, motion, and richer product and questionnaire tools on top of the initial mobile launch.

- **Native chrome** — Every tab uses iOS large titles with translucent blurred headers, a nav-bar search field on list screens, and a floating blurred tab bar that content scrolls under. Product creation opens in a native modal.
- **Native motion** — Buttons and rows respond with press-scale and highlight feedback. The segmented control slides between options, sheets use spring physics with drag-to-dismiss and keyboard avoidance, and tab switches include haptics and an icon pop. The home dashboard animates in with staggered entrances.
- **Products** — Rebuilt to match the web store manager. You can now create medication and regular products, assign storefront categories, configure plan menus with custom cadences, add variations and blocked states, and link a per-product intake questionnaire — all from a sectioned native editor backed by a shared catalog store.
- **Questionnaires** — Now a functional intake builder. Add answer options, define `showIf` branching between questions, manage sections, and reorder content. Questionnaires are available directly in the product link pickers.

No configuration is required — reinstall the app to pick up the update.

</Update>

<Update label="July 9, 2026">

## 🚀 New features

**Cuvo mobile staff console (iOS)**

The Cuvo staff console is now available as a native iOS app — a companion to the web console for staff who need to run the business from their phone. The mobile app covers the full breadth of the console across 45 screens:

- **Home & command center** — daily dashboard and the unified action queue
- **Messenger** — patient conversations and inbox
- **Patients & leads** — profiles, journeys, and lead pipeline
- **Orders, products & questionnaires** — catalog and clinical intake management
- **Payments suite** — transactions, payouts, disputes, subscriptions, and payment links
- **Analytics** — live traffic and reporting
- **Settings & integrations** — profile, members, security, billing, compliance, developers, notifications, appearance, and connected apps
- **Get started & auth** — onboarding and sign-in

At launch, the mobile app runs on fixture data so teams can explore the full surface before live data wiring lands. No configuration is required beyond installing the app — staff sign in with the same account they use on the web console.

</Update>

<Update label="July 4, 2026">

## 🚀 New features

**AI Website Studio**

Build and edit your site through a chat-driven experience. Ask for changes in plain language, preview them instantly, and publish when ready — no manual layout work required. Powered by Fireworks GLM-5.2.

**Multi-tenant workspaces**

Each brand now runs in its own isolated tenant with scoped drafts, publishing, and public rendering. Content and configuration stay separated across organizations by default.

**Block-based homepage builder**

Compose landing pages from a full library of ready-made blocks:

- Hero, Header, and Footer for site chrome
- FAQ and Blog Posts sections
- Benefits Marquee and Treatment Cards
- Product and Category cards
- Weight Loss Plan and CTA Banner
- BMI Calculator and Results carousel

Mix and match blocks to ship a complete 13-section homepage.

## ✨ Updates

- **Tenant host resolution** — Configured primary app hostnames now resolve to the correct default organization automatically.
- **Draft & publish workflow** — Tenant-scoped draft and publish controls let teams stage changes before they go live.
- **Public site rendering** — Published pages render on tenant-scoped public routes.

## 🐛 Bug fixes

- Content with empty fields now renders correctly instead of breaking the page.
- Tightened tenant isolation so records only appear in the workspace they belong to.
- Resolved a configuration loading issue that could prevent the app from starting in certain environments.

</Update>
