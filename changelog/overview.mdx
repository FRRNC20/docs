<Update label="July 10, 2026">

## 🚀 New features

**Patient identity verification**

Verify patient identity end-to-end from the Command Center. Each patient now has an **Identity** tab that surfaces:

- **Status card** — current decision (Approved, Declined, In Review, Not Started), last update, and the workflow the session ran through. Standalone verifications with no hosted workflow render as `—`.
- **Checks card** — individual identity checks (document authenticity, face match, liveness, and more) with per-check pass/fail state.
- **Document card** — extracted document details (type, number, issuing country, expiration).
- **Media strip** — the portrait, document front and back, liveness capture, and face-match frames. Images are served through short-lived presigned links from a private PHI bucket rather than the provider's live URLs.
- **Session history** — every verification attempt the patient has made, newest first.
- **Event log** — a chronological audit trail of state changes on the identity record.

**Identity step in Get Started**

The new-patient Get Started flow now includes a three-part identity step: personal details → document upload → review. Patients can complete verification inline before the rest of onboarding continues.

**ID portrait on the patient roster and console header**

The most recent Approved ID portrait now appears next to the patient on the roster and in the patient console header. Portraits are fetched as a single batched, presigned request per page. Patients without an Approved portrait fall back to initials.

## ✨ Updates

- **Standalone verifications supported** — Identity records without a hosted verification workflow are accepted; the workflow field renders as `—` wherever it appears.
- **PHI media hardening** — Provider-issued media URLs are stripped from the identity decision before it reaches the browser. Staff always view portrait, document, liveness, and face-match images through short-lived presigned links from the private PHI bucket.

</Update>

<Update label="July 4, 2026">

## 🚀 New features

**AI Website Studio**

Build and edit your site through a chat-driven experience. Ask for changes in plain language, preview them instantly, and publish when ready — no manual layout work required. Powered by Fireworks GLM-5.2.

**Multi-tenant workspaces**

Each brand now runs in its own isolated tenant with scoped drafts, publishing, and public rendering. Content and configuration stay separated across organizations by default.

**Block-based homepage builder**

Compose landing pages from a full library of ready-made blocks:

- Hero, Header, and Footer for site chrome
- FAQ and Blog Posts sections
- Benefits Marquee and Treatment Cards
- Product and Category cards
- Weight Loss Plan and CTA Banner
- BMI Calculator and Results carousel

Mix and match blocks to ship a complete 13-section homepage.

## ✨ Updates

- **Tenant host resolution** — Configured primary app hostnames now resolve to the correct default organization automatically.
- **Draft & publish workflow** — Tenant-scoped draft and publish controls let teams stage changes before they go live.
- **Public site rendering** — Published pages render on tenant-scoped public routes.

## 🐛 Bug fixes

- Content with empty fields now renders correctly instead of breaking the page.
- Tightened tenant isolation so records only appear in the workspace they belong to.
- Resolved a configuration loading issue that could prevent the app from starting in certain environments.

</Update>
