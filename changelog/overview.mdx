<Update label="July 8, 2026">

## 🚀 New features

**Realtime staff messenger**

The staff messenger now runs on a live realtime platform. Conversations update the instant something happens — no more waiting on the 15-second poll to see a new reply, a new thread, or an inbound patient SMS.

- **Instant thread updates** — Replies, new conversations, and inbound SMS appear in the thread as soon as they land, across every messenger view.
- **Live typing indicator** — When a teammate is typing on a patient thread, you see a typing bubble in real time; it clears as soon as they stop or send.
- **Smart scroll and unread pill** — The thread opens at the newest message and stays pinned to the bottom while you're there. If you've scrolled up to read history, new arrivals raise a "New message" pill instead of yanking the view — click it (or scroll to the bottom) to catch up.

No setup is required. The realtime experience is on by default for all staff users, and the messenger falls back to its previous polling behavior automatically if the realtime service is unavailable.

</Update>

<Update label="July 4, 2026">

## 🚀 New features

**AI Website Studio**

Build and edit your site through a chat-driven experience. Ask for changes in plain language, preview them instantly, and publish when ready — no manual layout work required. Powered by Fireworks GLM-5.2.

**Multi-tenant workspaces**

Each brand now runs in its own isolated tenant with scoped drafts, publishing, and public rendering. Content and configuration stay separated across organizations by default.

**Block-based homepage builder**

Compose landing pages from a full library of ready-made blocks:

- Hero, Header, and Footer for site chrome
- FAQ and Blog Posts sections
- Benefits Marquee and Treatment Cards
- Product and Category cards
- Weight Loss Plan and CTA Banner
- BMI Calculator and Results carousel

Mix and match blocks to ship a complete 13-section homepage.

## ✨ Updates

- **Tenant host resolution** — Configured primary app hostnames now resolve to the correct default organization automatically.
- **Draft & publish workflow** — Tenant-scoped draft and publish controls let teams stage changes before they go live.
- **Public site rendering** — Published pages render on tenant-scoped public routes.

## 🐛 Bug fixes

- Content with empty fields now renders correctly instead of breaking the page.
- Tightened tenant isolation so records only appear in the workspace they belong to.
- Resolved a configuration loading issue that could prevent the app from starting in certain environments.

</Update>
