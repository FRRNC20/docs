<Update label="July 7, 2026">

## 🚀 New features

**Products manager**

The new Products area at `/products` replaces the Treatments Center with a Shopify-style catalog experience. Manage your full product roster from a single roster view and open any product in a dedicated editor.

The list page includes:

- Roster view with search, type, category, status, and sort filters synced to the URL
- Price and States summary columns at a glance
- Active/inactive toggle directly on each row — flip a product live or back to draft without opening the editor
- Excluded-states info tooltip on the States column that lists blocked states by full name on hover

**Product editor**

Every product opens in a full-page editor at `/products/[id]`, and new products start from `/products/new`. The editor uses a responsive, capped-width layout built around the patient-page bar. A live snapshot strip shows the current price, plans, and state coverage, and a save bar tracks draft-vs-baseline changes.

Editor cards cover:

- **Identity, Media, Highlights, Organization, Storefront** — core product metadata, direct-to-S3 media upload, and site placement
- **Commerce** — headline price plus purchase-option plans, with per-plan pricing surfaced inline
- **Medication** — type-gated card for medication products
- **State availability** — 51-state grid with Enable/Disable all and a behavior toggle that either hides the product or shows it as unavailable in excluded states
- **Purchase options** — checkbox-driven plan selection (one-time and subscription cadences) with per-plan dollar fields and min-1/unique validation

**Variations and custom cadences**

Products now support multiple variations — each with its own name and its own purchase-option menu — so you can offer different strengths, pack sizes, or bundles under a single product. Alongside the standard preset cadences, you can add **custom cadences** using an "Every N &lt;unit&gt; · Add plan" control. Preset-equivalent intervals normalize back onto the preset row, and duplicates are rejected inline. The snapshot strip surfaces active plans plus a `· N variations` suffix so operators can see the full shape of a product at a glance.

## ✨ Updates

- **Treatments Center retired** — The legacy `/catalog` area and the Treatments Center navigation entry have been removed. Everything now lives under Products.
- **Archive and restore** — Products can be archived from the editor (behind a confirm) and restored later; archived rows in the list stay locked until you restore them from the editor.

</Update>

<Update label="July 4, 2026">

## 🚀 New features

**AI Website Studio**

Build and edit your site through a chat-driven experience. Ask for changes in plain language, preview them instantly, and publish when ready — no manual layout work required. Powered by Fireworks GLM-5.2.

**Multi-tenant workspaces**

Each brand now runs in its own isolated tenant with scoped drafts, publishing, and public rendering. Content and configuration stay separated across organizations by default.

**Block-based homepage builder**

Compose landing pages from a full library of ready-made blocks:

- Hero, Header, and Footer for site chrome
- FAQ and Blog Posts sections
- Benefits Marquee and Treatment Cards
- Product and Category cards
- Weight Loss Plan and CTA Banner
- BMI Calculator and Results carousel

Mix and match blocks to ship a complete 13-section homepage.

## ✨ Updates

- **Tenant host resolution** — Configured primary app hostnames now resolve to the correct default organization automatically.
- **Draft & publish workflow** — Tenant-scoped draft and publish controls let teams stage changes before they go live.
- **Public site rendering** — Published pages render on tenant-scoped public routes.

## 🐛 Bug fixes

- Content with empty fields now renders correctly instead of breaking the page.
- Tightened tenant isolation so records only appear in the workspace they belong to.
- Resolved a configuration loading issue that could prevent the app from starting in certain environments.

</Update>
