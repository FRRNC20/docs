<Update label="July 10, 2026">

## 🚀 New features

**Visual questionnaire builder**

The questionnaire builder is now a node-and-edge canvas. Drop question nodes onto the canvas, connect them with condition edges to branch on specific answers, and drag to reorder the served sequence. The old card/toolbox layout has been replaced.

**Draft and publish workflow**

Questionnaire edits now save to a **draft** that is separate from the version served to patients. Iterate freely, then click **Publish** to promote the draft to the live structure. Publishing runs the servability gate — if it fails, patients keep seeing the last published version.

**Live preview link**

A new **Live preview** panel in the builder shows a tokenized URL that renders your current draft inside the real patient app. Copy the link to open it on your phone, or keep the inline iframe open while you edit — it polls the draft and refreshes as you save. The link is minted on your first autosave.

**Per-product intake questionnaires**

The medication card in the product editor now has **Initial intake** and **Follow-up intake** pickers. Choose any enabled questionnaire per product, or leave either set to **Vertical default** to keep the product category's standard intake. Purchases of that product serve the linked questionnaire instead of the category default.

## ✨ Updates

- **Persistent app shell** — The sidebar, main layout, and call provider now stay mounted as you move between sections, so navigation is instant and in-flight state (open calls, unsaved panels) survives cross-section clicks.
- **Linked-but-disabled questionnaires stay visible** — If a product links to a questionnaire that has since been disabled, the picker still shows the link with an "unavailable" marker so you can re-point or reset it.

</Update>

<Update label="July 9, 2026">

## 🚀 New features

**Live questionnaire builder**

Staff edits in the questionnaire builder now persist to the database and become the source that serves patient intake flows. The mock save is gone — publish real changes to `/apply/*` funnels without a code deploy.

**Patient-eye preview**

The toolbar **Preview** button now opens an interactive preview of the current canvas from the patient's perspective, including:

- `showIf` visibility between questions
- Exclusive option behavior (for example, "None of the above")
- Non-blocking hard-stop warnings
- BMI composite input (weight in pounds, height in feet and inches)
- Consent statements

Use it to sanity-check a flow before saving.

## ✨ Updates

- **Servability gate on save** — The builder rejects structures that would not serve cleanly to patients. Blocked cases include unsupported question types, duplicate option labels, hard-stop or exclusive labels that do not match an option, `showIf` referencing a later question, missing BMI composite fields (`weightLbs`, `heightFt`, `heightIn`), and untrimmed labels or options.
- **Atomic option renames** — Renaming an option now propagates on blur to every reference in one step: the question's own hard-stop and exclusive settings, plus any other question's `showIf` value that pointed at it. In-flight keystrokes no longer trigger partial renames.
- **Preview visibility parity** — The preview's `notEquals` handling now matches the live serving logic, so steps that depend on an unanswered question stay visible in preview just as they do for patients.
- **Audit logging** — Questionnaire updates record the exact question IDs that changed.

</Update>

<Update label="July 4, 2026">

## 🚀 New features

**AI Website Studio**

Build and edit your site through a chat-driven experience. Ask for changes in plain language, preview them instantly, and publish when ready — no manual layout work required. Powered by Fireworks GLM-5.2.

**Multi-tenant workspaces**

Each brand now runs in its own isolated tenant with scoped drafts, publishing, and public rendering. Content and configuration stay separated across organizations by default.

**Block-based homepage builder**

Compose landing pages from a full library of ready-made blocks:

- Hero, Header, and Footer for site chrome
- FAQ and Blog Posts sections
- Benefits Marquee and Treatment Cards
- Product and Category cards
- Weight Loss Plan and CTA Banner
- BMI Calculator and Results carousel

Mix and match blocks to ship a complete 13-section homepage.

## ✨ Updates

- **Tenant host resolution** — Configured primary app hostnames now resolve to the correct default organization automatically.
- **Draft & publish workflow** — Tenant-scoped draft and publish controls let teams stage changes before they go live.
- **Public site rendering** — Published pages render on tenant-scoped public routes.

## 🐛 Bug fixes

- Content with empty fields now renders correctly instead of breaking the page.
- Tightened tenant isolation so records only appear in the workspace they belong to.
- Resolved a configuration loading issue that could prevent the app from starting in certain environments.

</Update>
